# 多模态推理服务 — 项目步骤清单

根据 `demand.txt` 整理的项目当前需要完成的步骤，按执行顺序排列。

---

## 一、项目基础与规范

- [ ] **1.1** 确认/搭建 Python 3.10 环境
- [ ] **1.2** 使用 **uv** 初始化项目并满足 uv 包管理规范（`pyproject.toml` 等）
- [ ] **1.3** 配置 **ruff** 做代码格式化，并（可选）配置为 pre-commit hook
- [ ] **1.4** 约定并执行：所有 Python 代码需有**明确类型标注**
- [ ] **1.5** 编写简单的**单元测试**，保证核心逻辑可测

---

## 二、服务框架与接口形态

- [ ] **2.1** 使用 **FastAPI** 搭建服务，同时支持 **HTTP** 和 **WebSocket** 两种接口形式（具体路径与报文格式自定）
- [ ] **2.2** 接口层统一使用 **asyncio**，采用 **async** 语义（async/await）
- [ ] **2.3** 设计并实现：服务**单次启动时只支持一种能力**（文生图 **或** 图片理解二选一），可通过配置/环境变量切换

---

## 三、接口与模型实现

### 文生图（Text-to-Image）

- [ ] **3.1** 实现「文生图」接口：输入文字，返回图片；单次启动若选文生图则只暴露此接口
- [ ] **3.2** 使用 **HunyuanDiT 1.5B**，通过 **diffusers/transformers** 加载
- [ ] **3.3** 手动调用 **processor**，理清：原始输入 → processor 处理 → 模型输入的数据流，并在代码/注释中体现

### 图片理解（Vision-Language）

- [ ] **3.4** 实现「图片理解」接口：输入图片 + 文本，输出文本；单次启动若选图片理解则只暴露此接口
- [ ] **3.5** 使用 **Qwen2.5VL-3B**，通过 **transformers** 加载
- [ ] **3.6** 同样理清 processor 与数据流：原始输入 → 预处理 → 模型输入

---

## 四、模型缓存与部署

- [ ] **4.1** 模型文件通过 **HuggingFace 环境变量**（如 `HF_HOME` / `TRANSFORMERS_CACHE` 等）缓存到宿主机
- [ ] **4.2** 编写 **Dockerfile**，将上述缓存目录通过 **Docker 挂载** 挂进容器，避免每次构建重复下载
- [ ] **4.3** 确保在 Linux 或 Windows WSL2 下可正常调起容器并完成一次端到端测试

---

## 五、请求排队与并发控制

- [ ] **5.1** 实现**请求排队**：同一模型**不并发**处理多个请求，多请求时按队列顺序逐个推理
- [ ] **5.2** 在接口层（HTTP/WebSocket）与推理层之间增加队列层，保证「单 worker 单模型」语义

---
